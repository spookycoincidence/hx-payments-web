# 🧾 Convenciones y buenas prácticas

## Código

- HTML semántico
- Comentarios breves en las secciones clave
- Inputs accesibles con `label` asociado

## CSS

- Estilos definidos en `css/style.css`
- Clases en inglés con nombres consistentes: `.btn`, `.input-group`, etc.
- Evitar `!important`

## JS

- Código plano en `main.js`
- Documentación mínima en funciones (si se agregan)

## Naming

- Archivos en minúscula y con guiones (`send-money.html`)
- Clases BEM opcional si se escala
